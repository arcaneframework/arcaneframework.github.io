template<typename T>
class Arcane::UniqueArray< T >

Vecteur 1D de données avec sémantique par valeur (style STL).

Cette classe gère un tableau de valeur de la même manière que la classe stl::vector de la STL.

La sémantique par valeur fonctionne comme suit:

UniqueArray<int> a1(5);
UniqueArray<int> a2;
a2 = a1; // a2 devient une copie de a1.
a1[3] = 1;
a2[3] = 2;
std::cout << a1[3]; // affiche '1'

Definition arccore/src/collections/arccore/collections/Array.h:1878

Il est possible de spécifier un allocateur mémoire spécifique via le constructeur UniqueArray(IMemoryAllocator*). Dans ce cas, l'allocateur spécifié en argument doit rester valide tant que cette instance est utilisée.

Avertissement
L'allocateur est transféré à l'instance de destination lors d'un appel aux constructeurs qui prennent en argument un Array, SharedArray ou UniqueArray. Il en est de même avec l'opérateur d'assignement et lors de l'appel à UniqueArray::swap(). Si ces appels sont envisagés, il faut garantir que l'allocateur restera valide même après transfert. Il est donc préférable dans tout les cas que l'allocateur spécifique utilisé reste valide durant toute la durée de l'application.

Si le type est un type Plain Object Data (POD) alors les données ne sont pas initialisées en cas de réallocation. La classe template ArrayTraits permet de spécifier si un type est POD suivant la valeur données par le type ArrayTraits<T>::IsPODType qui peut être FalseType ou TrueType. Sauf spécialisation, seuls les types de base du C++ sont POD.
