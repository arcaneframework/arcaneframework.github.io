'arcane-post-traitement' module
Options concernant les sorties pour le dépouillement.
Les sorties peuvent être effectués en fonction du nombre d'itérations (options periode-sortie) ou en fonction du temps physique (option frequence-sortie). Par exemple:
<periode-sortie>5</periode-sortie>

Detailed list of options

compression-sortie-historique (bool=false) 
<compression-sortie-historique>value</compression-sortie-historique>

Evite de sauvegarder tous les points d'abcisse entre les périodes de l'option output-history-period. Cela permet de faire baisser la consomation mémoire notamment lorsque le code effectue un grand nombre d'itérations.

depouillement

Liste des variables et des groupes à dépouiller.

depouillement/groupe [0... undefined] (Arcane::ItemGroup) 
<groupe>value</groupe>

Groupe d'entité de maillage sur lequel on souhaite exporter des variables. Il est possible de mettre plusieurs options de ce type pour exporter les variables sur plusieurs groupes. Si cette option n'est pas spécifiée, les variables sont sorties sur l'ensemble du maillage.

depouillement/variable [0... undefined] (ustring) 
<variable>value</variable>

Nom d'une variable à exporter. Il est possible de mettre plusieurs options de ce type pour exporter plusieurs variables. Si aucune variable n'est spécifiée, aucune sortie n'est effectuée.

format-service (Arcane::IPostProcessorWriter=Ensight7PostProcessor) 
<format-service name='value'>service configuration block</format-service>

Possible values for tag name:
EnsightHdfPostProcessor
Ensight7PostProcessor
Indique le type de bibliothèque à utiliser pour les sorties de dépouillement.

frequence-sortie (real=0.0) 
<frequence-sortie>value</frequence-sortie>

Temps physique entre deux sorties. Si cette valeur vaut zéro, aucune sortie n'est effectuée. Si cette option est présente avec l'option periode-sortie, cette dernière n'est pas prise en compte.

periode-sortie (integer=0) 
<periode-sortie>value</periode-sortie>

Nombre d'itérations entre deux sorties. Si cette valeur vaut zéro, aucune sortie au cours des itérations n'est effectuée.

periode-sortie-historique (integer=1) 
<periode-sortie-historique>value</periode-sortie-historique>

Nombre d'itérations entre deux sorties pour les historiques. Si cette valeur vaut zéro, les sorties pour les historiques sont désactivées.

sauvegarde-initiale (bool=false) 
<sauvegarde-initiale>value</sauvegarde-initiale>

Si vrai, effectue une sortie en fin d'initialisation.

sauvegarde-temps-final (bool=true) 
<sauvegarde-temps-final>value</sauvegarde-temps-final>

Si vrai, effectue une sortie en fin d'exécution lorsque le temps final est atteint.

sortie-fin-execution (bool=false) 
<sortie-fin-execution>value</sortie-fin-execution>

Si vrai, effectue une sortie de dépouillement en fin d'exécution. La fin d'exécution n'est pas forcément identique au temps final en cas d'utilisations de protections/reprises ou si on spécifie un nombre maximum d'itérations lors de l'exécution.
