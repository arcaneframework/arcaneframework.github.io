114/*!
115 * \brief Interface du gestionnaire de traces.
116 *
117 Une instance de cette classe gère les flots de traces.
118 Pour envoyer un message, il suffit d'appeler la méthode corrrespondante
119 (info() pour un message d'information, error() pour une erreur, ...)
120 pour récupérer un flot et d'utiliser l'opérateur << sur ce flot pour
121 transmettre un message.
122
123 Par exemple:
124 \code
125 ITraceMng* tr = ...;
126 tr->info() << "Ceci est une information.";
127 int proc_id = 0;
128 tr->error() << "Erreur sur le processeur " << proc_id;
129 \endcode
130
131 Le message est envoyé lors de la destruction du flot. Dans les
132 exemples précédents, les flots sont temporairement créés (par la méthode info())
133 et détruits dès que l'opérateur << a été appliqué dessus.
134
135 \warning Il faut absolument appeler la méthode finishInitialize() avant
136 d'utiliser les appels aux méthodes pushTraceClass() et popTraceClass().
137
138 Si on souhaite envoyer un message en plusieurs fois, il faut stocker
139 le flot retourné:
140
141 \code
142 TraceMessage info = m_trace_mng->info();
143 info() << "Début de l'information.\n"
144 info() << "Fin de l'information.";
145 \endcode
146
147 Il est possible d'utiliser des formatteurs simples sur les messages
148 (via la classe #TraceMessage)
149 ou des formatteurs standards des iostream en appliquant l'opérateur operator()
150 de TraceMessage.
151
152 Les instances de cette classe sont gérées par compteur de référence. Il
153 est préférable de conserver les instances dans un ReferenceCounter.
154 */
