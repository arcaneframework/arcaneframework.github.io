26/*!
27 * \brief Informations pour envoyer/recevoir un message point à point.
28 *
29 * Il existe deux manières de construire une instance de cette classe:
30 *
31 * 1. en donnant un couple (rang destinataire,tag). Le tag est optionnel
32 * et s'il n'est pas spécifié, sa valeur sera celle de MessageTag::defaultTag().
33 * 2. via un MessageId obtenu lors d'un appel à mpProbe(). Dans ce dernier
34 * cas, l'instance ne peut alors être utilisée qu'en réception via mpReceive().
35 *
36 * Il est possible de spécifier si le message sera bloquant lors de la
37 * construction ou via l'appel à setBlocking(). Par défaut un message
38 * est créé en mode bloquant.
39 *
40 * L'émetteur (emiterRank()) du message est l'émetteur et la
41 * destination (destinationRank() le récepteur. Pour un message d'envoi (mpSend()),
42 * destinationRank() est donc le rang de celui qui va recevoir le message. Pour un
43 * message de réception (mpReceive()), destinationRank() est le rang de celui dont
44 * on souhaite recevoir le message ou A_NULL_RANK si on souhaite recevoir de
45 * n'importe qui.
46 *
47 * \note L'émetteur est en général positionnée par l'implémentation de
48 * IMessagePassingMng car il correspond au rang de celui qui poste le message.
49 * L'utilisateur n'a donc jamais besoin de le spécifier.
50 */
