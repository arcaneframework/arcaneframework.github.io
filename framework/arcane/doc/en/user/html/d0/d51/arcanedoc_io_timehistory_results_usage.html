We have two types of files here: the "summary" and the curve files generated by writers.

The time_history.json file (and time_history.xml, kept for compatibility reasons) is a JSON format "summary." It contains all information about value histories such as the history name, the support, the source subdomain, and a unique name. It is generated regardless of which writers are enabled.

Furthermore, two writers are active by default (they implement the Arcane::ITimeHistoryCurveWriter2 interface):

  • ArcaneCurveWriter
  • GnuplotTimeHistoryCurveWriter2

Their documentation is available in the Arcane developer documentation.

GnuplotTimeHistoryCurveWriter2

This writer generates all files contained in the gnuplot folder. Each file is in the form:

1.7385346701095061E-01 6.3790000000000001E-03
3.4770693402190123E-01 6.5709999999999996E-03
5.2156040103285184E-01 6.1009999999999997E-03

Each line designates an iteration. Two columns: the first column contains the simulation time, the second column designates the history value.

For reading these files, using the "summary" can be useful since these files contain no additional information.

ArcaneCurveWriter

This writer generates the curves.acv file. This file is an Arcane format and contains all information about all curves. The "summary" file is therefore not necessarily useful for this format.

To utilize this file, we can use the arcane_curves program, found in install_dir/bin/arcane_curves.
