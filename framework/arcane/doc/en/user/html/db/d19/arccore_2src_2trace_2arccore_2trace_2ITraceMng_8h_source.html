133/*!
134 * \brief Trace manager interface.
135 *
136 * An instance of this class manages trace streams.
137 * To send a message, simply call the corresponding method
138 * (info() for an information message, error() for an error, ...)
139 * to retrieve a stream and use the << operator on this stream to
140 * transmit a message.
141 *
142 * For example:
143 * \code
144 * ITraceMng* tr = ...;
145 * tr->info() << "This is an information message.";
146 * int proc_id = 0;
147 * tr->error() << "Error on processor " << proc_id;
148 * \endcode
149 *
150 * The message is sent upon destruction of the stream. In the
151 * previous examples, the streams are temporarily created (by the info() method)
152 * and destroyed as soon as the << operator has been applied to them.
153 *
154 * \warning It is absolutely necessary to call the finishInitialize() method before
155 * using the calls to the pushTraceClass() and popTraceClass() methods.
156 *
157 * If you want to send a message in several parts, you must store
158 * the returned stream:
159 *
160 * \code
161 * TraceMessage info = m_trace_mng->info();
162 * info() << "Start of information.\n"
163 * info() << "End of information.";
164 * \endcode
165 *
166 * It is possible to use simple formatters on messages
167 * (via the #TraceMessage class)
168 * or standard iostream formatters by applying the operator() operator
169 * of TraceMessage.
170 *
171 * Instances of this class are managed by a reference counter. It
172 * is preferable to keep instances in a ReferenceCounter.
173 */
